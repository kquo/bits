# Style Guide

## Purpose

This file defines the editorial style for `tips`.

## Voice Or Style

- Personal, authentic, honest, reflective, reasoned, accurate when possible, and open to criticism and revision

## Audience

- Primarily the repo owner, as a durable personal reference.
- Secondarily anyone who is shared a link and wants a concise, honest statement of the current view.

## Editorial Model

- Treat each page as a durable reference note, not a stream-of-consciousness draft or news post.
- Prefer short sections, lists, and examples when they improve scanability.
- Keep the repo markdown-first. Avoid adding tooling-specific front matter unless the site actually needs it.

## Category Routing

- `Life`: practical living, health, hobbies, entertainment, household matters, and other non-technical day-to-day guidance.
- `Mind`: reasoning, epistemology, philosophy, psychology of thinking, cognitive biases, and mental models.
- `Society`: politics, economics, institutions, culture, discourse, and collective behavior.
- `Tech`: software, hardware, programming, cloud, networking, devices, and digital tooling.
- Route by the reader's primary intent. If the dominant question is "how do I use or fix technology?" put it in `Tech`. If it is "how should I think?" put it in `Mind`. If it is about public systems or group behavior, put it in `Society`. Otherwise prefer `Life`.
- Pick one landing area only. If a tip could plausibly fit elsewhere, record the alternate in review notes rather than publishing duplicates.

## Entry Standards

- Use a clear page title that matches the label shown from the relevant index page.
- Open with one or two lines that state the page's scope before diving into details.
- Prefer concrete claims, examples, and distinctions over filler.
- Keep HTML and heavy formatting limited. Plain Markdown is the default unless an existing page already relies on HTML for a specific reason.
- Add external links only when they improve orientation or support a non-obvious claim.
- When a trusted general-reference link fits naturally, prefer a small number of [Wikipedia](https://en.wikipedia.org/wiki/Main_Page) links as the default first source.
- Do not overlink. One to three natural anchor links per short section is usually enough.

## Editorial Checks

- prefer accuracy over speed
- fact-check claims when practical, and push back on weak, unsupported, or overstated assertions
- keep claims supportable and reviewable
- keep terminology consistent across published content
- check new or revised entries against related pages across the site and resolve contradictions or vague tensions
- tighten wording, remove duplication, and fix grammar before publish
- verify internal links, filenames, and index references when adding or moving content
- make opinionated statements read as opinionated statements, not as settled facts
